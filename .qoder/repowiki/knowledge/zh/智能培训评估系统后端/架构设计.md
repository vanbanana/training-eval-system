- **分层架构**：采用经典的 `handler` (HTTP层) -> `service` (业务逻辑) -> `repository` (数据访问) -> `store` (SQLite封装) 结构。
- **异步流水线**：核心评估流程由 `internal/pipeline` 编排，通过 `internal/worker` 线程池异步执行文档解析、LLM 评分和相似度比对任务。
- **实时通信**：使用 `internal/sse` 实现 Server-Sent Events 广播，向客户端推送处理进度和通知。
- **依赖注入**：在 `cmd/server/main.go` 中集中初始化所有组件并组装路由，通过 `internal/handler/router.go` 定义 API 边界与中间件链（鉴权、限流、CORS）。
- **数据存储**：基于 `modernc.org/sqlite` 实现纯 Go 的 SQLite 访问，支持 WAL 模式与读写分离连接池。
