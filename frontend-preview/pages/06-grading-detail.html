1实验目的

通过实现一个基于阻塞队列的生产者-消费者模型，加深对 Java 多线程并发原语（synchronized、wait/notify）的理解，并掌握使用 ReentrantLock 与 Condition 重构同步代码的方法，理解 signal、await 在条件变量上的语义。

2设计思路与关键代码

使用一个固定容量的 ArrayBlockingQueue 作为缓冲，生产者按指数分布间隔产生订单 push 至队列，消费者拉取队列后按平均处理时间消费。

AI 核查发现关键逻辑点
报告中通篇使用了 ReentrantLock + Condition 实现同步原语，但部分代码片段中 ProducerConsumer.java 仍有 synchronized + wait/notify，建议在课堂中明确两者关系。
public synchronized void produce(int item) { while (queue.size() == capacity) { wait(); } queue.add(item); notifyAll(); }
3性能测量与分析

在 4 生产者 + 4 消费者场景下，缓冲容量从默认的 8000 ops/s 左右，至加倍后曾试过为 9870 ops/s，经过性能瓶颈在 wait/notify 的唤醒/挂起 1230 次。
